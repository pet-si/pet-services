## **Sobre**

O programa PET Services é desenvolvido por alunos da **Universidade Federal de Santa Maria**, do curso de **Sistemas de Informação**, integrantes do **Programa de Ensino Tutorial (PET)**, com o time de desenvolvimento originalmente composto por Mateus Cardoso, Gabriel Saueressig e Michel Altmann.

A proposta de desenvolvimento de software trata-se de um sistema de gerenciamento para pequenos negócios locais e prestadores de serviço de Santa Maria. O software deve ser desenvolvido de forma incremental, com o apoio de empreendedores participantes da pesquisa de mercado.

---

## **Pesquisa de mercado**

A primeira etapa desenvolvida com o público foi a pesquisa de mercado, com o objetivo de levantar requisitos e ambientar o grupo no mundo dos negócios. A pesquisa foi realizada utilizando a plataforma [Google Forms](https://docs.google.com/forms) e abordou temas como: perfil do negócio, perfil do usuário, dispositivos disponíveis, dificuldades atuais do negócio e funcionalidades desejadas.

---

### Resultados alcançados

**Perfil de negócio**: a maioria se enquadra como profissional autônomo ou prestador de serviços, e uma parcela menor como e-commerce ou loja física.

**Perfil de cliente**: mais de 80% dos pesquisados acreditam não ter dificuldade com o uso de tecnologia.

**Dispositivos disponíveis**: há uma predominância significativa de dispositivos Android e Windows, com maioria no setor *mobile*. Uma parcela dos dispositivos móveis utiliza *iOS*. No total, os celulares representam cerca de 60%.

**Dificuldades atuais**: duas dificuldades destacam-se das demais: divulgação, que não é abrangida pelo projeto, e controle financeiro, no qual o sistema pode auxiliar. Além disso, também há dificuldades relacionadas a relatórios, controle de vendas, gestão de estoque, cadastro de clientes, entre outras.

**Funcionalidades desejadas**: além das funcionalidades diretamente ligadas às dificuldades, foram recebidas demandas por vendas/catálogo online, integração com *WhatsApp*, portal do cliente e agendamento online. Tais funcionalidades fogem do escopo social e gratuito do projeto.

**Conclusão**: o desenvolvimento deve ser *mobile-first*. Há uma diversidade de áreas de atuação, portanto o projeto deve possuir uma base genérica e customizável.

---

## Padrões adotados

**Modelo de desenvolvimento:** Incremental com foco em código limpo, revisão e refatoração

**Escrita:** Além dos padrões convencionais, optou-se por utilizar camelCase para frontend e snake_case para o banco de dados; o idioma escolhido foi inglês; a forma de identação segue uma separação visual por blocos lógicos.
